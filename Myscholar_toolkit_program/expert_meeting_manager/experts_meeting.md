# Experts Meeting：多学科虚拟专家协同科研探索流程

## 概述

本文档记录了一套**基于多学科虚拟专家协同的交互式科研建模工作流**。该流程的核心思想是：针对一个需要多学科交叉的复杂研究问题，同时召集多个领域的虚拟专家，让每个专家从自身学科视角独立进行文献调研和方法学分析，最终通过结构化综合形成统一的建模方案。

整个过程可在单次会话中完成，形成从"问题定义 → 并行调研 → 多学科综合 → 可视化呈现"的完整闭环。

---

## 阶段 1：问题定义（Problem Definition）

### 输入
- 一个**跨学科的研究问题**，涉及多个知识领域的交叉
- 明确的**实验设计/数据生成机制**描述
- 清晰的**核心科学问题**（我们想知道什么？）

### 关键要素
| 要素 | 说明 |
|------|------|
| 研究对象 | 描述实验的参与者、刺激和任务 |
| 变量体系 | 定义自变量（操纵条件）、因变量（观测反应）和潜在变量（待推断） |
| 数据格式 | 明确数据的结构：条件数 × 采样点数 × 被试数 × 试次数 |
| 科学目标 | 用一句话表述核心推断目标 |

### 输出
一份简洁的问题陈述文档，包含实验设计表、变量定义和核心科学问题。

---

## 阶段 2：专家团队组建（Team Assembly）

### 原则
根据研究问题涉及的**学科维度**，选择 6-10 个相关领域的专家。每个专家应具有独特且互补的视角。

### 专家选择框架

| 维度 | 典型学科示例 | 提供的视角 |
|------|-------------|-----------|
| **理论层** | 基础科学（如物理学、心理学、生物学） | 核心理论框架和机制解释 |
| **方法层** | 统计学、机器学习、数学建模 | 数学规格、推断方法和可识别性分析 |
| **工程层** | 工程学、计算机科学、信号处理 | 计算实现、算法设计和效率优化 |
| **应用层** | 临床医学、人因工程、交互设计 | 生态效度、实际操作约束和应用价值 |
| **验证层** | 神经科学、生理学 | 生物/物理约束和机制验证 |

### 输出
一份专家清单，每个专家附有：
- 学科名称和子方向
- 在该问题中的角色定位
- 需要调研的具体子话题列表

---

## 阶段 3：并行专家调研（Parallel Expert Survey）

### 工作模式
所有专家**同时、独立**开展工作。每个专家收到相同的"问题陈述"和针对其学科的"调研指令"。

### 专家调研指令模板

```
你是 [学科领域] 的专家。请针对以下研究场景撰写调研文档：

[问题陈述]

请从你的学科视角出发，覆盖以下内容：
1. 该领域与本研究相关的核心理论/模型/框架
2. 关键方法学工具和技术
3. 高水平SCI期刊中的相关关键论文（含具体标题、作者、年份）
4. 针对本实验设计的推荐建模方法
5. 与其他学科可能的关联点

要求：SCI期刊水平，含 LaTeX 数学公式，Markdown 格式
```

### 并行执行
- 使用异步并行机制同时启动所有专家
- 每个专家独立工作，互不知晓其他专家的内容
- 这确保了**真正的多学科独立视角**，避免群体思维

### 输出
每个专家一个独立的 Markdown 调研文档，包含该学科的：
- 理论框架和关键概念
- 数学模型和方法
- 关键参考文献
- 针对该问题的建模建议
- 与其他学科的潜在关联

---

## 阶段 4：综合与建模（Synthesis & Modeling）

### 4.1 交叉分析
- 通读所有专家文档
- 识别**跨学科共识**（多个学科独立得出相同结论的模型/方法）
- 识别**互补视角**（不同学科提供同一问题的不同侧面）
- 识别**独特贡献**（某一学科独有的方法或约束）

### 4.2 统一建模
基于跨学科共识，构建**统一的数学模型框架**：

1. **核心模型**：被多学科独立验证的基础模型结构
2. **层级扩展**：从简单到复杂的模型谱系
3. **数学规格**：完整的似然函数、先验分布和后验推断方法
4. **可识别性分析**：参数可唯一估计的条件
5. **模型比较策略**：候选模型的系统比较框架

### 4.3 综合文档结构

```
1. 问题重述（精简版）
2. 跨学科共识（核心发现）
3. 核心模型规格（含完整数学定义）
4. 替代与补充模型（模型谱系）
5. 模型比较策略（方法×准则矩阵）
6. 可识别性与功效分析
7. 跨学科洞察（每个学科对核心模型的贡献）
8. 推荐分析流程（从数据到结论的完整路径）
```

### 输出
一份综合结论文档，整合所有专家视角，形成统一的建模方案。

## 阶段 5：交互式可视化（Interactive Visualization）

### 设计原则
- **单文件部署**：一个 HTML 文件包含所有内容，无需服务器
- **公式正确渲染**：使用 MathJax 或 KaTeX 渲染 LaTeX
- **关系可视化**：Canvas/SVG 绘制学科-模型关系网络
- **分页导航**：总体综述 + 各专家详细视角

### 可视化组件

| 组件 | 内容 | 技术 |
|------|------|------|
| 研究总览 | 实验设计、关键数据统计 | HTML + CSS |
| 学科关系网络 | 以核心模型为中心的放射状网络图 | Canvas 2D |
| 核心模型详解 | 模型方程、多学科根基卡片 | MathJax + CSS Grid |
| 模型谱系图 | 候选模型的层级关系 | Canvas 2D |
| 分析流程管道 | 从数据到结论的步骤流程 | CSS Flexbox |
| 专家详情页 | 每个专家的关键贡献和与核心模型的联系 | 分区内容 |

### 交互设计
- 左侧固定导航栏
- 核心模型相关条目用特殊颜色标记
- 专家卡片可点击跳转到详情

### 输出
一个自包含的交互式 HTML 文件。

---

## 阶段 6：多语言输出（Localization）

### 原则
- 所有内容文档提供中英双语版本
- 翻译保持学术水准，LaTeX 公式、代码块、引用标题原样保留
- 并行批量翻译以提高效率

### 翻译规则

| 保留原样 | 需要翻译 |
|----------|---------|
| LaTeX 数学公式（$$...$$ 和 $...$） | 叙述性文本 |
| 代码块（Stan、Python 等） | 章节标题 |
| 参考文献中的论文标题 | 表格内容 |
| 标准技术术语（如 Kalman filter、von Mises） | 图注和表注 |

### 输出
与源文件平行的中文版本，目录结构一致。

---

## 阶段 7：增强与迭代（Enhancement & Iteration）

### 7.1 公式说明增强
在所有公式下方添加变量说明行，提升文档的可读性和教学价值。

**格式：**
- 英文：`> **Where:** var1 — description1; var2 — description2`
- 中文：`> **其中：** 变量1 — 说明1；变量2 — 说明2`

### 7.2 综合平台构建
将所有子文档和可视化整合为一个**综合交互平台**：
- 总览页面展示全局结构和关键数据
- 核心模型页面突出跨学科共识
- 关系网络图展示学科间的验证和互补结构
- 各专家详情页提供深入视角
- 模型对比页展示完整谱系和比较策略

### 输出
- 增强后的所有源文档（含公式变量说明）
- 综合交互平台 HTML

## 工具链与效率策略

### 并行化架构

```
                     ┌── 专家 1 (学科A) ──→ 调研文档1.md
                     ├── 专家 2 (学科B) ──→ 调研文档2.md
问题定义 + 专家清单 ──┼── 专家 3 (学科C) ──→ 调研文档3.md
                     ├── 专家 4 (学科D) ──→ 调研文档4.md
                     ├── ...          ──→ ...
                     └── 专家 N (学科N) ──→ 调研文档N.md
                                              │
                            ┌─────────────────┘
                            ▼
                    综合 + 建模 ──→ conclusion.md
                                      │
                            ┌─────────┼─────────┐
                            ▼         ▼         ▼
                      可视化.html  翻译批次1-4  综合平台.html
```

### 关键效率原则

| 原则 | 说明 |
|------|------|
| **并行优先** | 所有独立任务同时启动，不分先后 |
| **独立调研** | 专家互不知晓，避免群体思维，确保真正的多学科独立视角 |
| **统一模板** | 专家指令、翻译规则、公式说明均使用统一格式 |
| **增量增强** | 先完成基础版本，再统一添加增强（如公式说明） |
| **自包含输出** | 可视化使用单文件 HTML，无外部依赖 |
| **双语输出** | 中英并行，扩大受众范围 |

### 质量保证

- **汇聚效度**：多学科独立推导出相同模型 → 强汇聚效度
- **区分效度**：不同学科提供独特的约束和验证角度
- **完整性**：从理论→方法→实现→应用→验证的完整覆盖
- **可复现性**：完整的数学规格和实现代码（Stan 模型）

## 适用场景

本工作流适用于以下类型的研究问题：

- 需要**多学科交叉**的复杂建模问题
- 存在明确的**实验设计**和**数据生成机制**
- 核心目标是**推断潜在变量**（权重、参数、结构）
- 理论框架需要**多个独立视角的交叉验证**
- 需要产出一套**完整的建模文档**（从理论到实现）

典型领域：
- 认知科学中的多感觉整合建模
- 生物医学中的多模态数据融合
- 社会科学中的多层次因果推断
- 工程中的多物理场耦合建模
- 环境科学中的多源数据同化

## 迭代改进方向

1. **专家交互轮次**：在第一轮独立调研后，让专家互相阅读对方的文档并进行第二轮评论
2. **自动化增强**：公式变量说明、交叉引用链接等可通过脚本自动化
3. **实时协作**：多专家同时在线讨论，而非完全异步
4. **数据驱动验证**：将真实数据纳入流程，对比各模型的实证表现
5. **扩展输出格式**：除 HTML 外，支持生成 LaTeX 论文草稿、Jupyter Book、交互式 Dashboard
